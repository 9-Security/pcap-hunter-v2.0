# PCAP Hunter

**PCAP Hunter** 是一個進階的、AI 增強的威脅獵捕工作台，旨在填補手動封包分析與自動化資安監控之間的鴻溝。它賦予 SOC 分析師和威脅獵捕人員從原始 PCAP 檔案中快速攝取、分析並提取可執行情資的能力。

透過結合業界標準的網路分析工具 (**Zeek**, **Tshark**) 與現代 **大型語言模型 (LLMs)** 及 **OSINT** APIs，PCAP Hunter 自動化了封包分析中繁瑣的部分——解析、關聯和豐富化——讓分析師能專注於偵測與回應。

---

## 🚀 關鍵功能

### 1. 🧠 AI 驅動的威脅分析
- **自動化報告**: 生成專業、SOC 就緒的報告，總結關鍵發現、可疑指標和風險評估。
- **本地與雲端 LLM 支援**:
  - **本地隱私**: 完全相容於 [LM Studio](https://lmstudio.ai/) (例如 Llama 3, Mistral)，適合隔離網段或是對隱私敏感的分析環境。
  - **雲端運算**: 支援 OpenAI 相容 API，以利用如 GPT-4 等更大型的模型。
- **多語言支援**: 使用地區特定的術語生成多種語言的報告 (英文、繁體/簡體中文、日文、韓文、義大利文、西班牙文、法文、德文)。
- **感知上下文**: AI 會接收網路流、Zeek log 和 OSINT 資料的結構化摘要，充當專家的副駕駛。
- **MITRE ATT&CK 對應**: 自動將行為和指標對應到 MITRE ATT&CK 框架和攻擊鏈階段。
- **攻擊敘事合成**: 專業的威脅分析，將原始事件轉化為連貫、可執行的故事。
- **專屬分析分頁**: LLM 報告和敘事被隔離在專屬的「LLM Analysis」分頁中，以便專注查看。

### 2. 🎯 IOC 優先級評分
- **自動化風險評估**: 使用加權評分引擎動態排名指標 (嚴重、高、中、低)。
- **多因子分析**: 分數源自 OSINT 信譽 (VirusTotal, AbuseIPDB, GreyNoise)、行為模式 (C2 信標) 和鑑識上下文 (DGA, JA3 惡意軟體比對)。

### 3. 📁 案件管理系統
- **調查工作區**: 儲存發現、管理觀察項目並追蹤資安事件的進度。
- **證據追蹤**: 將相關流量和殘留物直接存入調查案件中。

### 4. 🖨️ 專業 PDF 匯出
- **利害關係人就緒**: 生成格式化、專業的 PDF 報告，包含完整的攻擊敘事、技術摘要和儀表板視覺化。

### 6. 🔍 深度封包檢測與流量分析
- **多引擎管道**: 使用 **PyShark** 進行細顆粒封包檢測，並使用 **Tshark** 進行高速統計。
- **協定解析**: 自動提取並視覺化主要協定的元數據：
  - **HTTP**: 方法、URI、User-Agents。
  - **DNS**: 查詢、回應、記錄類型。
  - **TLS/SSL**: 伺服器名稱 (SNI)、憑證詳細資訊。
  - **SMB**: 檔案共享和指令。

### 7. 🛡️ Zeek 整合
- **自動化生命週期**: 管理上傳 PCAP 上的 Zeek 執行，無需手動指令列介入。
- **Log 分析**: 解析並關聯核心 Zeek log 至互動式資料表：
  - `conn.log`: 連線摘要與狀態。
  - `dns.log`: 名稱解析活動。
  - `http.log`: 網頁流量細節。
  - `ssl.log`: 加密流量元數據。

### 8. 🕵️ 進階 DNS 與 TLS 鑑識
- **DNS 分析**:
  - **DGA 偵測**: 使用夏農熵 (Shannon Entropy) 識別網域生成演算法。
  - **DNS Tunneling**: 偵測高流量/異常 DNS 負載。
  - **Fast Flux**: 識別解析為快速變化 IP 位址的網域。
- **TLS 指紋識別**:
  - **JA3/JA3S**: 計算並比對 SSL/TLS 指紋與已知惡意軟體資料庫。
  - **憑證分析**: 驗證憑證鏈並偵測自簽或過期憑證。

### 9. 📡 C2 信標 (Beaconing) 偵測
- **啟發式分析**: 實作統計演算法以偵測命令與控制 (C2) 信標行為。
- **評分引擎**: 基於以下項目對流量進行排名：
  - **週期性**: 通訊間隔的規律性 (低變異數)。
  - **抖動 (Jitter)**: C2 代理程式的隨機化嘗試。
  - **傳輸量**: 負載大小的一致性。

### 10. 📦 負載提取 (Carving) 與鑑識
- **YARA 掃描器**: 使用自訂 YARA 規則掃描提取的檔案，立即識別惡意軟體。
- **檔案提取**: 使用 `tshark` 從流量中提取 HTTP 檔案本體。
- **殘留物雜湊**: 自動計算提取檔案的 SHA256 雜湊值以進行信譽檢查。
- **安全儲存**: 提取的檔案儲存在本地隔離目錄中以供手動分析。

### 11. 🌍 互動式世界地圖與儀表板
- **全球能見度**: 在大型互動式世界地圖上視覺化流量來源與目的地。
- **流量大小**: 線條粗細基於連線量變化，凸顯主要資料流。
- **交叉篩選**:
  - **統一鑽取**: 在任何圖表 (地圖、圓餅圖或時間軸) 中選取資料會篩選整個儀表板。
  - **協定篩選**: 點擊協定圓餅圖中的切片以隔離該協定。
  - **時間篩選**: 在流量時間軸上選取範圍以專注於特定時間視窗。
- **持久檢視選項**:
  - **排除私有 IP**: 儀表板頂端的專用切換開關，允許在分析中忽略本地流量。此設定在互動探索期間保持有效。
- **重置功能**: 包含「清除所有篩選」按鈕，可輕鬆重置儀表板視圖，包括私有 IP 切換。

### 12. 📊 整合式 TopN 與指標
- **聚合視圖**: 用於 Top IP、Port、協定和網域的水平長條圖和表格。
- **封包長度分佈**: 視覺化所擷取流量的負載大小分佈。

### 13. 🌐 OSINT 情資豐富化
與領先的威脅情資提供商整合以驗證入侵指標 (IOCs)：
- [x] 📖 **[使用手冊](./USER_MANUAL.md)**: 附有截圖的詳細功能指南。
- **VirusTotal**: 檔案雜湊與 IP/網域信譽。
- **AbuseIPDB**: 社群回報的 IP 濫用報告。
- **GreyNoise**: 識別網際網路背景雜訊與掃描器。
- **OTX (AlienVault)**: 開放威脅交換脈衝與指標。
- **Shodan**: 面向網際網路的裝置詳細資訊與開放 Port。
- **智慧快取**: SQLite 支援的快取系統，可節省 API 配額並加速重複查詢。

## 🛠️ 安裝

### 先決條件
- **Python 3.10+**
- **Zeek**: `brew install zeek` (macOS) 或透過套件管理員 (Linux)。
- **Tshark**: `brew install wireshark` (macOS) 或 `sudo apt install tshark` (Linux)。
- **Pango**: `brew install pango` (macOS) - 產生 PDF 報告所需 (WeasyPrint)。
- **LM Studio** (選用): 用於本地 LLM 推論。

### 快速開始
1. **Clone 儲存庫**:
   ```bash
   git clone https://github.com/ninedter/pcap-hunter.git
   cd pcap-hunter
   ```
2. **安裝依賴套件**:
   ```bash
   make install
   ```
3. **執行應用程式**:
   ```bash
   make run
   ```

## 📖 使用指南

1. **上傳 (Upload)**: 
   - 將 `.pcap` 檔案拖放到 **Upload** 分頁。
2. **設定 (Configure)**:
    - 設定您的 LLM Endpoint (預設: `http://localhost:1234/v1`)。
    - **自家位置 (Home Location)**: 使用級聯的 洲 -> 國家 -> 城市 選擇器設定您的地理位置，以獲得準確的地圖連線視覺化。
    - 新增 OSINT 服務的 API 金鑰 (選用但建議)。
    - 切換特定分析階段 (例如啟用/停用 "OSINT Cache", "Carving")。
    - **重新執行報告**: 如果您更改語言或模型，點擊 "Re-run Report" 僅重新生成 LLM 報告，而無需重新處理 PCAP。
    - **資料管理**: 使用 Config 分頁中細緻的 "Clear" 按鈕分別清除 PCAP 資料、OSINT 快取或整個案件資料庫。
3. **分析 (Analyze)**: 點擊 **Extract & Analyze**。
4. **監控 (Monitor)**: 觀察 **Progress** 分頁中的管道執行：
   - *封包計數* -> *解析* -> *Zeek* -> *DNS/JA3* -> *信標偵測* -> *Carving* -> *OSINT* -> *報告*。
5. **審閱 (Review)**: 閱讀生成的 **威脅報告** 並探索原始資料表以進行深入分析。

## ⚙️ 設定
預設值在 `app/config.py` 中管理。關鍵設定包括：
- `DATA_DIR`: 儲存分析產物的對象 (預設: `./data`)。
- `OSINT_CACHE_ENABLED`: SQLite 快取開關 (預設: True)。
- `DEFAULT_PYSHARK_LIMIT`: 最大深度解析封包數 (預設: 200,000)。
- `OSINT_TOP_IPS_DEFAULT`: 豐富化的前幾名活躍者數量 (預設: 50)。
